🔷 What is a Key in SQL?
A key is used to identify records uniquely and maintain relationships between tables.



🔑 1. Primary Key

👉 Definition:
A column (or set of columns) that uniquely identifies each row.

👉 Rules:
Cannot be NULL
Must be unique
Only ONE primary key per table

👉 Example idea:
StudentID in a student table



🔑 2. Foreign Key

👉 Definition:
A column that creates a relationship between two tables.

👉 Rules:

Can have duplicate values
Can be NULL (depends on design)
References primary key of another table

👉 Example idea:
Order table → CustomerID refers to Customer table



🔑 3. Candidate Key

👉 Definition:
All possible columns that can uniquely identify a record.

👉 Important:
One of them becomes the Primary Key

👉 Example idea:
Email, Aadhaar number (both unique → candidate keys)



🔑 4. Super Key

👉 Definition:
A set of one or more columns that uniquely identifies a row.

👉 Important:

Can contain extra unnecessary columns
Every candidate key is a super key

👉 Example idea:
StudentID OR (StudentID + Name)



🔑 5. Alternate Key

👉 Definition:
Candidate keys that are NOT selected as primary key.

👉 Example idea:
If Email is not primary, it becomes alternate key



🔑 6. Composite Key

👉 Definition:
A primary key made using multiple columns.

👉 Example idea:
(StudentID + CourseID) in enrollment table



🔑 7. Unique Key

👉 Definition:
Ensures all values are unique.

👉 Difference from Primary Key:

Can allow one NULL (in most DBs)
Can have multiple unique keys



🔑 8. Surrogate Key

👉 Definition:
Artificial key generated by system.

👉 Example idea:
Auto-increment ID

👉 Use case:
When no natural key exists



🔑 9. Natural Key

👉 Definition:
Real-world meaningful data used as key.

👉 Example:
Email, Phone number



🔑 10. Secondary Key

👉 Definition:
Used for searching/filtering but not unique.

👉 Example:
City column


🔥 Easy Way to Remember (Interview Trick)

👉 PK = Identity
👉 FK = Relationship
👉 Candidate = Options
👉 Super = Combination
👉 Alternate = Backup
👉 Composite = Multiple columns
